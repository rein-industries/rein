# Rein — mobile

**Your coding agents, anywhere.**

This is the public issue tracker for the **Rein mobile app**. It holds no source
code — just a place to report bugs and request features. The app itself is
developed privately.

## Report a bug

The fastest path is from inside the app: **Settings → Help → Report a bug**. That
opens a new issue with your app version, protocol version, and device already
filled in, so we can reproduce it without a back-and-forth.

You can also [open a bug report here](https://github.com/rein-industries/mobile/issues/new?template=bug_report.yml).

## Request a feature

[Open a feature request](https://github.com/rein-industries/mobile/issues/new?template=feature_request.yml)
and tell us the problem you're trying to solve.

## Before you open an issue

- **Search [existing issues](https://github.com/rein-industries/mobile/issues)** —
  someone may have already reported it. A 👍 on an existing issue helps us
  prioritize.
- **One issue per report.** Separate bugs and ideas are easier to track and fix.
- **Never include secrets.** Don't paste pairing tokens, access tokens, or
  anything from a session transcript. We never need your code to fix a bug.

## Privacy

Rein is built so your code and conversations stay between your phone and your own
machines. Please keep it that way in your reports — screenshots and steps are
plenty.
